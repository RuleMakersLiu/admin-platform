# 智能体状态同步机制设计

## 1. 概述

### 1.1 设计目标
- **实时性**: 状态变更延迟 < 500ms
- **一致性**: 确保所有客户端看到的状态一致
- **可靠性**: 消息不丢失,支持断线重连后恢复
- **扩展性**: 支持6个智能体,可扩展到更多

### 1.2 技术架构
```
智能体进程 → HTTP API → MySQL → Observer → Redis Pub/Sub → WebSocket Server → 客户端
```

## 3. 状态同步流程

### 3.1 智能体主动上报状态

#### 心跳机制
```
智能体 → 每30秒 → POST /api/v1/agents/{id}/heartbeat
```

**请求参数:**
```json
{
  "status": "working",
  "current_task_id": "uuid",
  "workload": 75,
  "progress": 60,
  "metadata": {
    "cpu_usage": 45,
    "memory_usage": 60
  }
}
```

**服务端处理:**
```php
public function heartbeat(Request $request, $agentId)
{
    // 1. 更新数据库
    $agent = Agent::find($agentId);
    $agent->update([
        'status' => $request->status,
        'current_task_id' => $request->current_task_id,
        'workload' => $request->workload,
        'last_heartbeat_at' => now(),
    ]);

    // 2. 更新Redis缓存
    Redis::hmset("agent:{$agentId}", $agent->toArray());

    // 3. 触发状态变更事件
    event(new AgentStatusUpdated($agent));

    return response()->json(['success' => true]);
}
```

### 3.2 任务触发状态变更

#### 任务分配时
```
任务分配 → 智能体状态: idle → working
```

```php
// TaskAssignmentService.php
public function assignTask($taskId, $agentId)
{
    DB::transaction(function () use ($taskId, $agentId) {
        // 1. 创建任务关联
        TaskAgent::create([
            'task_id' => $taskId,
            'agent_id' => $agentId,
            'status' => 'assigned',
        ]);

        // 2. 更新智能体状态
        $agent = Agent::find($agentId);
        $agent->update([
            'status' => 'working',
            'current_task_id' => $taskId,
        ]);

        // 3. 创建协作记录
        CollaborationRecord::create([
            'task_id' => $taskId,
            'agent_id' => $agentId,
            'action' => 'started',
        ]);
    });
}
```

#### 任务完成时
```
任务完成 → 智能体状态: working → idle
```

### 3.3 任务转交触发状态变更

```
智能体A转交任务给智能体B
→ A: working → idle (如果无其他任务)
→ B: idle → working
```

```php
public function transferTask($taskId, $fromAgentId, $toAgentId)
{
    DB::transaction(function () use ($taskId, $fromAgentId, $toAgentId) {
        // 1. 更新任务关联
        TaskAgent::where('task_id', $taskId)
            ->where('agent_id', $fromAgentId)
            ->update(['status' => 'completed', 'completed_at' => now()]);

        TaskAgent::create([
            'task_id' => $taskId,
            'agent_id' => $toAgentId,
            'status' => 'working',
            'started_at' => now(),
        ]);

        // 2. 记录转交动作
        CollaborationRecord::create([
            'task_id' => $taskId,
            'agent_id' => $fromAgentId,
            'action' => 'transferred',
            'details' => ['to_agent_id' => $toAgentId],
        ]);

        CollaborationRecord::create([
            'task_id' => $taskId,
            'agent_id' => $toAgentId,
            'action' => 'started',
        ]);

        // 3. 更新智能体状态
        $this->updateAgentStatusAfterTransfer($fromAgentId);
        $this->updateAgentStatusAfterTransfer($toAgentId);
    });
}

private function updateAgentStatusAfterTransfer($agentId)
{
    $agent = Agent::find($agentId);
    $activeTasks = TaskAgent::where('agent_id', $agentId)
        ->where('status', 'working')
        ->count();

    if ($activeTasks > 0) {
        $agent->update([
            'status' => 'working',
            'workload' => min(100, $activeTasks * 50),
        ]);
    } else {
        $agent->update([
            'status' => 'idle',
            'workload' => 0,
            'current_task_id' => null,
        ]);
    }
}
```

## 4. 状态推送机制

### 4.1 触发式推送

#### Observer监听
```php
// AgentObserver.php
class AgentObserver
{
    public function updated(Agent $agent)
    {
        // 只在状态变更时触发
        if ($agent->isDirty(['status', 'workload', 'current_task_id'])) {
            // 1. 发布到Redis频道
            Redis::publish('agent_updates', json_encode([
                'agent_id' => $agent->id,
                'name' => $agent->name,
                'status' => $agent->status,
                'workload' => $agent->workload,
                'current_task' => $agent->currentTask ? [
                    'id' => $agent->currentTask->id,
                    'title' => $agent->currentTask->title,
                    'progress' => $agent->currentTask->progress,
                ] : null,
                'timestamp' => now()->toIso8601String(),
            ]));
        }
    }
}
```

### 5.2 Redis分布式锁

```php
// 防止并发更新冲突
$lock = Redis::set("lock:agent:{$agentId}", 1, 'NX', 'EX', 5);

if (!$lock) {
    throw new \Exception('Agent is being updated by another process');
}

try {
    // 更新智能体状态
} finally {
    Redis::del("lock:agent:{$agentId}");
}
```

## 6. 断线重连处理

### 6.1 客户端重连

```javascript
// 客户端WebSocket连接
class AgentDashboard {
    connect() {
        this.ws = new WebSocket('ws://server:9501?token=xxx');
        
        this.ws.onopen = () => {
            // 订阅房间
            this.ws.send(JSON.stringify({
                type: 'subscribe',
                room: 'dashboard'
            }));
            
            // 请求全量数据
            this.requestFullSync();
        };
        
        this.ws.onclose = () => {
            // 5秒后重连
            setTimeout(() => this.connect(), 5000);
        };
    }
    
    requestFullSync() {
        // 通过HTTP API获取全量数据
        fetch('/api/v1/agents')
            .then(res => res.json())
            .then(data => {
                this.renderAgents(data.agents);
            });
    }
}
```

## 8. 性能优化

### 8.1 减少推送频率

```php
// 使用防抖机制
class AgentStatusUpdateDebouncer
{
    private $pending = [];
    private $timer;

    public function push($agentId, $data)
    {
        $this->pending[$agentId] = $data;
        
        // 100ms后批量推送
        if (!$this->timer) {
            $this->timer = Timer::after(100, function () {
                $this->flush();
            });
        }
    }

    private function flush()
    {
        foreach ($this->pending as $agentId => $data) {
            Redis::publish('agent_updates', json_encode($data));
        }
        $this->pending = [];
        $this->timer = null;
    }
}
```

### 8.2 增量更新

```php
// 只推送变更的字段
public function updated(Agent $agent)
{

        'agent_id' => $agent->id,
        'changes' => $changes,  // 只包含变更的字段
        'timestamp' => now()->toIso8601String(),
    ]));
}
```

## 9. 6个智能体协作示例

### 9.1 完整工作流

```
1. PM(项目经理) 创建任务
   → PM: idle → working
   
2. PM 分配任务给 需求分析师
   → PM: working → idle
   → 需求分析师: idle → working
   
3. 需求分析师完成任务,转交给 UI设计师
   → 需求分析师: working → idle
   → UI设计师: idle → working
   
4. UI设计师完成设计,转交给 前端工程师 + 后端工程师
   → UI设计师: working → idle
   → 前端工程师: idle → working
   → 后端工程师: idle → working
   
5. 前后端完成开发,转交给 测试工程师
   → 前端工程师: working → idle
   → 后端工程师: working → idle
   → 测试工程师: idle → working
   
6. 测试通过,任务完成
   → 测试工程师: working → idle
   → 任务: completed
```

### 9.2 并行协作

```
任务: 开发用户管理模块

并发状态:
- 前端工程师: working (开发用户列表组件)
- 后端工程师: working (开发用户API)
- 测试工程师: working (编写测试用例)

此时3个智能体同时working
```

## 12. 总结

### 核心机制
1. **心跳保活**: 30秒间隔,90秒超时
2. **事件驱动**: 状态变更通过Observer触发推送
3. **Pub/Sub**: Redis频道解耦业务层和WebSocket层
4. **定时刷新**: 看板每5秒全量更新
5. **断线重连**: 客户端自动重连+全量同步

### 技术要点
- Swoole协程提高并发性能
- Redis保证实时性和可靠性
- 数据库事务保证一致性
- WebSocket实现双向通信
- Observer模式实现解耦
